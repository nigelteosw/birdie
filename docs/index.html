Birdie

Local-first mentorship capture

Turn before/after edits into reviewed team lessons.

Birdie is a Claude Code plugin and MCP server that captures real edits, lets the assistant extract candidate lessons, keeps humans in the review loop, and answers questions from approved examples.

Install Birdie See the workflow

What it does

A compact review loop for mentorship knowledge.

01

Capture the edit

Store the before text, after text, contributor names, role context, and optional playbook guidance.

02

Extract the lesson

The MCP host does the reasoning. Birdie validates the quote and saves the result for review.

03

Promote with review

Review, edit, reject, or promote lessons before they become part of the searchable teaching set.

Install

Use Birdie as a Claude Code plugin.

The published plugin ships with a bundled MCP server for macOS, Linux, and Windows. Plugin users do not need to clone the repository, run bun install, or start a separate server. 

/plugin marketplace add nigelteosw/birdie
/plugin install birdie

First run

Start a new Claude Code conversation and ask Birdie to set itself up. If you do not have a shared team server URL, choose local mode.

Local mode

State stays under your home directory.

Config

~/.birdie/config.json

Stores whether Birdie should use local storage or a shared remote server.

Database

~/.birdie/birdie.db

Stores captured traces, candidate lessons, review status, and promoted examples.

Domain profile

~/.birdie/domain.md

Optionally customizes the domain, typology, and mentorship-worthy guidance.

Workflow

Capture, review, then query.

1. Capture a before/after edit

Ask Birdie to capture an edit as a mentorship example. Include who submitted it, who it affected, and why the edit mattered.

2. Extract a candidate lesson

Run the extraction prompt for the trace. The assistant proposes the quote, change, rationale, typology, and playbook alignment.

3. Review the queue

Birdie opens the review UI on demand, usually at http://127.0.0.1:6677. It reuses an existing Birdie server on that port or falls back to a free port when needed.

4. Ask from reviewed examples

Ask how a senior handled a similar situation, or ask what a junior is struggling with across promoted lessons.

Development

Work on Birdie locally.

bun install
bun run build
bun run test

The plugin only needs the MCP server. Run the REST API or web UI directly when you are iterating outside the plugin flow. 

bun run dev:backend -- web
bun run dev:backend -- mcp
